# Safety Policy สำหรับ Jarvis ใช้กับเครื่องจริง

## หลักการ

Jarvis ควรช่วยทำงาน แต่ไม่ควรมีสิทธิ์ทำลายไฟล์ ส่งข้อความ หรือเปลี่ยนระบบโดยไม่ถามยืนยัน

## คำสั่งที่ต้องถามยืนยันเสมอ

- ลบไฟล์/โฟลเดอร์
- ย้าย/เขียนทับไฟล์สำคัญ
- รัน terminal หรือ shell command
- ส่งข้อความผ่าน WhatsApp/Telegram/Email
- กด submit ฟอร์มราชการ/การเงิน/ข้อมูลส่วนบุคคล
- restart/shutdown/lock เครื่อง
- ติดตั้งโปรแกรม
- แก้ system settings

## วิธีตอบเมื่อคำสั่งเสี่ยง

ตัวอย่าง:

```text
คำสั่งนี้จะลบไฟล์จริง: C:\Users\...\report.xlsx
ยืนยันให้ลบไหม? ตอบว่า "ยืนยันลบ" เท่านั้นผมถึงจะทำต่อ
```

## หลักการใช้งานกับข้อมูล วพอ.

- ห้ามส่งรายชื่อนักเรียน/บุคลากรออกเว็บภายนอกโดยไม่จำเป็น
- งาน Excel/รายงานควรประมวลผล local ก่อน
- ไฟล์ที่มีข้อมูลส่วนบุคคลควรแยกจาก repository
- API key ต้องอยู่ใน `config/api_keys.json` หรือ environment variable เท่านั้น
